# 项目说明

这是我的个人作品集网站项目，我正在从零开始学习编程。
我没有编程基础，请用最简单的方式和我沟通。

技术栈：HTML + CSS + JavaScript

# 对话规则

- 所有回答和代码注释必须使用中文
- 修改代码前先解释你打算怎么做，等我说"好的"再动手
- 每次修改后告诉我：改了哪些文件、改了什么、为什么这么改
- 如果不确定我的需求，先问我，不要猜测
- 我问"为什么"的时候，请用非程序员能理解的语言解释

# 代码规范

- 每个函数上方必须写中文注释，说明这个函数的作用
- 变量名使用英文，采用小驼峰命名（如 userName、orderCount）
- 变量名必须有明确含义，禁止使用 a、b、x 等无意义名称
- 每个代码文件不超过 200 行，超过了要拆分
- HTML 标签要正确缩进，保持层次清晰

# 工作流程

- 一次只做一个小任务，不要一次性改太多东西
- 写完代码后，主动进行一轮自查，检查有没有 bug
- 告诉我如何运行和测试你写的代码
- 如果引入了新概念，先用一句话解释是什么、为什么需要它

# 禁止事项

- 不要引入新的第三方库，除非我同意
- 不要删除已有的代码注释
- 不要使用高级语法（如装饰器、元编程等），保持代码简单
- 不要一次性生成超过 100 行的代码，分步来
- 不要跳过错误处理，每个可能出错的地方都要处理

# CSS 代码规范

- 颜色统一使用十六进制写法（如 #1a73e8），不要用英文单词（如 blue）
- 关键样式块上方必须写注释说明用途
- 优先使用 classList 切换样式，减少直接操作 style

# JavaScript 代码规范

- 判断相等必须用 ===，判断不等必须用 !==，禁止使用 == 和 !=
- 禁止使用 var，只使用 let 和 const
- 数组操作使用 push/splice 等内置方法，不要手动拼接
- 事件监听统一使用 addEventListener，不要用 onclick

# 常见错误记录（持续更新）

- 2026-03-25 | PowerShell 中文乱码 | 执行 [Console]::OutputEncoding = [System.Text.Encoding]::UTF8
- 2026-03-25 | cd Desktop 找不到路径 | Windows 桌面路径要用 cd ~\Desktop 或直接用绝对路径
